Frequently Asked Questions


Q:I found a broken link or bad data file, how do I report it?

A:Use the "Feedback" tab on the side of the page.

Q:What is FITS?

A: FITS is a transport format originally designed for moving observational data between different computer architectures. Many FITS files are, structurally at least, compliant with PDS4 data structure requirements, but care must be taken to ensure that the FITS structures are not being abused, or that logically distinct data objects are not being packed into a single data structure, which would negate PDS4 compliance.

Q:If there is both a PDS3 and PDS4 version of the data, which should I use?

A:We recommend using PDS4 as it might have more meta data available. However, the data themselves are the same in PDS3 and PDS4

Q:How do you search in the PDS?

A:There are multiple search tools available for PDS, search tools are available on the individual node pages (Atmospheres, Small Bodies etc). One the main PDS page there is a link to search all the PDS nodes. https://pds.nasa.gov/datasearch/data-search/

Q: How can I find all the data available on a given asteroid or comet?

A:Try the Ferret

Q: I'm working with an asteroid which was visited by a spacecraft. How can I select image data on a specific area or crater of the asteroid?

A:Try the Small Bodies Image Browser (SBIB)

Q: I can't find the data I need. Help!

A:Use the archive front page to find data by mission, by data type, or by target type. If you still can't find what you need, send a request through the Feedback tab on the side of the page and we will help you.

Q: I found the table of data I need but how do I find out what the columns mean or what the units are on the columns?

A: Each data file has a separate label file with the same filename, but with extension either .lbl or .xml. Open this file to find the column definitions and other information about the data file. Once downloaded you might want to use the PDS4 Viewer which displays the column headers with the data

Q:Why can't you archive normal image format like JPG and png?

A:JPG and PNG are compressed formats, and are therefore not suitable for archiving. JPEG is particularly unsuited for archiving, since it is a lossy format, meaning that data is lost on the first encode, and further processing degrades the image further.

Q:Can you archive TIFF?

A:Some TIFFs can be archived. The TIFF must be uncompressed, among other requirements. (Link to the TIFF standard)

Q:How can I validate my FITS files?

A:Fits compliant validation or PDS4 compliant validation (they are not the same.) For fits compliant validation the NASA FITS verifier is a useful tool.

Q:What metadata is useful to be put into a Dictionary?

A: Definitions created by a data preparer, an XML Schema file and an XML Schematron file used by software for label validation, a human-readable document in which the definitions, specifications, and constraints have been abstracted to facilitate use by human developers

Q:How good must my documentation be?

A:We archive for 50 years. Needs to describe things clear enough to let someone use/validate it without being able to ask questions.

Q:When do you use a Mission Dictionary vs. Discipline Dictionary?

A:A data defines the meaning and structure of classes and attributes. When used in the context of XML files, it can also constrain the content within an XML element. A discipline Dictionary is intended to provide a place for scientists to write definitions for the SBN discipline dictionary attributes.
