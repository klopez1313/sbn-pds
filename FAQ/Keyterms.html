FITS: FITS is a data format using an ASCII keyword-value header on a binary or ASCII image or table file. It is often used in PDS archiving because it can be PDS4 compatible and is a commonly used format in the astronomy and planetary science community. See https://fits.gsfc.nasa.gov/fits_standard.html for the full FITS specification.

IMG format:IMG format refers to binary files with the .img filename extension that store raw disk images of floppy disks, hard drives, and optical discs or a bitmap image

PDS4 dictionary:A PDS4 dictionary defines the meaning and structure of classes and attributes. When used defines the meaning and structure of classes and attributes. When used in the context of XML files, it can also constrain the content within an XML element.

PDS4 Discipline Dictionary: A discipline data dictionaries contain the classes and attributes specific to various disciplines

Project Dictionary: Project Dictionary is a dictionary containing classes and attributes specific to data archived for a given mission

NAIF, Kernels, SPICE: These data sets contain time-series details of the position and pointing of the spacecraft, and the position and rotation state of the planets/small bodies. It also includes exact definitions of the spacecraft equipment and instruments, to include how they are oriented on the spacecraft. The complete SPICE information provides deterministic information to compute relative and inertial positions, pointing and angles of objects in space. For more info, see NAIF at https://naif.jpl.nasa.gov/naif/.

XML:XML is an Extensible Markup Language (XML) file. These are really just plain text files that use custom tags to describe the structure and other features of the data or document. In PDS4, each data file or document has an XML label to describe it.
